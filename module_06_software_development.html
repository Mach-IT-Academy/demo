Lesson 3: The Five Fundamental Data Types

Data Types: Every piece of data stored in a program has a type that tells the computer what kind of value it is and how much memory to allocate. The five fundamental data types are: char (one character), string (zero or more characters), integer (whole number), float (decimal number), and Boolean (true or false).

📋 Scenario

A developer at Mach Financial Services is building a new account registration form. Before writing any code, she must decide what data type to assign to each field: the customer's first name, account number, current balance, whether the account is active, and the customer's status tier letter. Choosing the wrong data type causes errors — storing a balance as an integer, for example, would silently drop the cents.

The Five Fundamental Data Types

🔤 char

A single character

Stores exactly one letter, digit, or symbol. Often written in single quotes.

char grade = 'A' char initial = 'J' char symbol = '$' Size: typically 1 byte

📝 string

Zero or more characters

A sequence of characters — could be a word, sentence, or even empty. Often written in double quotes.

string name = "John Smith" string email = "j@mach.com" string empty = "" Size: variable

🔢 integer (int)

A whole number — no decimals

Used for counts, IDs, quantities, and any whole-number value. Attempting to store 3.7 as an int stores 3.

int age = 34 int accountID = 1048576 int itemCount = 0 Size: typically 4 bytes

🔣 float

A number with decimals

Used for money, measurements, percentages — any value that requires a fractional component. "Float" is short for floating-point.

float balance = 4827.93 float taxRate = 0.0825 float pi = 3.14159 Size: typically 4 bytes

✅ Boolean

True or False — only two possible values

Used for yes/no decisions, flags, and on/off states. The backbone of all conditional logic.

bool isActive = true bool isLoggedIn = false bool hasPermission = true Size: 1 byte (or 1 bit)

Common Data Type Mistakes

// WRONG: Storing a balance as an integer drops the cents int balance = 4827.93; // Stored as 4827 — cents silently lost! // CORRECT: Use float for money float balance = 4827.93; // Stored correctly as 4827.93 // WRONG: Storing a name as a char char name = "Smith"; // Error — char holds ONE character only // CORRECT: Use string for names string name = "Smith"; // Correct — string holds multiple characters // WRONG: Storing true/false as a string string isActive = "true"; // Works, but wastes memory and complicates logic // CORRECT: Use Boolean for true/false states bool isActive = true; // Efficient and semantically correct

✅ Concept Check

Question: A program needs to store whether a network port is open or closed. Which data type is most appropriate?

A) integer — store 1 for open, 0 for closed
B) string — store "open" or "closed"
C) Boolean — store true for open, false for closed
D) char — store 'O' or 'C'
Explanation: A Boolean is purpose-built for exactly two states — true or false. Open/closed, yes/no, on/off, active/inactive — all of these are Boolean conditions. While you could use integer (1/0) or string ("open"/"closed"), Boolean is semantically correct, memory-efficient, and integrates directly with conditional logic (IF isOpen THEN...). It's the right tool for the job.

Programming languages are grouped by how they execute and what they are designed to do. This exercise covers six fundamental categories. Understanding the defining characteristic of each category is the key to answering every scenario correctly. Collapse this panel when you are ready to test yourself.

ASM
Assembly Language
Low-Level · Hardware-Specific
Type: Translated to native machine code by an assembler — one instruction ≈ one CPU operation; no runtime layer
Key trait: Developer manually manages registers and memory — maximum control, minimal abstraction
Strengths: Absolute speed; smallest executable footprint; direct hardware register access; no OS required
Used for: Bootloaders, firmware, device drivers, OS kernels, exploit development, reverse engineering
Watch for: Scenarios involving firmware, registers, CPU-level control, or hardware-specific programming → Assembly
CPL
Compiled Language
Pre-Execution Translation · High Performance
Type: Source → compiler → executable binary or bytecode — code cannot run without a compile step first
Output: Native .exe / binary (C/C++) or platform-neutral bytecode (.class for Java, run on a JVM)
Strengths: Fast runtime performance; type errors caught at compile time; optimized machine-level output
Used for: Operating systems, enterprise applications, game engines, financial systems, mobile apps
Watch for: Scenarios requiring a build step, .exe output, or high-performance production software → Compiled
INT
Interpreted Language
Runtime Translation · Flexible & Portable
Type: Interpreter reads and executes source code line-by-line at runtime — no compile step required
Deployment: Source code is the deliverable — target machine must have the interpreter installed to run it
Strengths: Rapid development cycle; easy to read; highly portable; dynamic typing; interactive REPL
Used for: Data science, automation, web backends, rapid prototyping, scientific computing, AI/ML
Watch for: Scenarios with no build step, edit-and-run workflows, or languages like Python, Ruby, PHP → Interpreted
MKP
Markup Language
Structure & Presentation · Not Executable
Type: NOT a programming language — no logic, variables, or control flow; parsed and rendered, never executed
Syntax: Tag-based: <tagname attribute="value">content</tagname> — wraps content with descriptive annotations
Strengths: Human-readable; universal browser support; separates content from logic; easy to learn
Used for: Web pages, email templates, data interchange (XML), document formatting, configuration files
Watch for: Scenarios involving web structure, tags, content description, or non-executable formatting → Markup
SQL
Query Language
Database Interaction · Data Retrieval & Manipulation
Type: Declarative — statements sent to a DBMS engine; you specify WHAT you want, not HOW to retrieve it
Key trait: Purpose-built for database communication — not general-purpose; always used within a DBMS context
Strengths: Industry standard; joins across tables; powerful aggregation; transactions with ACID guarantees
Used for: Business intelligence, reporting, application backends, data warehouses, analytics, auditing
Watch for: Scenarios involving database retrieval, INSERT/UPDATE/DELETE operations, or structured data → Query
SCR
Scripting Language
Automation · Task Execution · Interpreted at Runtime
Type: Interpreted at runtime by a shell or engine — scripts run as plain-text source files, no build required
Key trait: Automates tasks by controlling other programs and OS resources — sequences commands rather than building apps
Strengths: Rapid automation; native OS integration; minimal boilerplate; easy to modify and re-run
Used for: System administration, CI/CD pipelines, IT automation, log processing, scheduled tasks, DevOps
Watch for: Scenarios involving OS task automation, admin scripts, or Bash/PowerShell/Perl workflows → Scripting

Concept What It Does Common Keywords Key Characteristic
Branching Directs execution down different paths based on a condition — only one path runs at a time if / elif / else / switch / case / ternary (?:) A condition is evaluated; the result determines WHICH block of code runs. The other blocks are skipped.
Constant A named value that is set once and cannot be changed during execution const / final / #define / Set-Variable -Option Constant / ALL_CAPS convention The value is fixed at declaration. Any attempt to reassign it causes an error. Used for values that should never change (port numbers, tax rates, limits).
Looping Repeats a block of code multiple times — either a set number of times or until a condition is met for / foreach / while / do-while / until The same block of code executes repeatedly. Each pass is called an iteration. The loop continues until the exit condition is satisfied.
Variable A named storage location whose value can be read and changed during execution let / var / int / string / $name / name= (assignment operator =) The value can change at any point during execution. A variable holds the current state and is updated as the program runs.

💡 Tip: The key distinction between Constant and Variable is mutability — a constant never changes after it is set; a variable can change at any time. The key distinction between Branching and Looping is repetition — branching runs a block ONCE based on a condition; looping runs a block REPEATEDLY until a condition is met.

What is Bash?
An interpreted scripting language AND a command-line shell
Bash — the Bourne Again Shell — is classified as both an interpreted scripting language and a command-line shell. Like PowerShell on Windows, it serves both roles simultaneously — you can type commands interactively or save them into a script file that runs automatically.
Bash is the default shell on most Linux systems and macOS. It requires no installation — it is already present on virtually every server in the world. That ubiquity is its defining advantage: a Bash script written today runs on any Linux machine tomorrow without modification.
Bash excels at file system operations, text processing, process management, and system administration. It is the glue language of Linux — combining powerful built-in tools like grep, awk, sed, and cron into automated workflows with minimal code.

Backing Up Patient Records to a Timestamped Archive
Patient records must be backed up daily. A Bash script copies the entire records directory to a timestamped archive folder — automatically, without anyone manually locating and copying files. The timestamp ensures every backup is uniquely named and never overwrites the previous one.
Copy and paste the script below into the terminal, and press Enter on your keyboard to view the result.
bash
#!/bin/bash SOURCE="/var/data/patient-records" BACKUP_DIR="/mnt/backup/patient-records" TIMESTAMP=$(date +"%Y-%m-%d_%H-%M-%S") DEST="${BACKUP_DIR}/${TIMESTAMP}" echo "Starting backup: $(date)" echo "Source : $SOURCE" echo "Dest : $DEST" mkdir -p "$DEST" cp -r "$SOURCE/." "$DEST/" echo "Backup complete: $(du -sh "$DEST" | cut -f1) copied." echo "Archive: $DEST"
What just happened — line by line
#!/bin/bash
The shebang line — tells the system which interpreter to use. Every Bash script begins with this line. Without it, the system would not know to run the file as a Bash script.
$(date)
Command substitution — runs the date command and inserts its output directly into the variable. The timestamp becomes part of the folder name, making every backup uniquely identifiable without any manual naming.
mkdir -p
Creates the destination directory and any missing parent directories. The -p flag prevents an error if the directory already exists — the script is safe to run repeatedly.
cp -r
Copies the entire directory recursively. Every file and subdirectory inside the source is copied to the destination — a complete snapshot of the records at that moment in time.
Bash strengths here
  • No dependencies — cp, mkdir, and date ship with every Linux system
  • Timestamp generated automatically — no human input needed
  • Script is five lines of real work — extremely concise
Bash limitations here
  • cp copies everything — rsync is better for incremental backups
  • No built-in encryption — sensitive records need additional tools
  • No error notification if the backup fails silently
Bash is the fastest path from zero to a working backup script on Linux. No libraries, no installation, no configuration — just a file and the shell that is already there.

Enabling PowerShell Scripts Across the Entire Network

An administrator needed every computer at Mach Regional Medical Center to allow PowerShell scripts to run automatically. Instead of walking to each workstation, he ran a single script from his own machine that pushed the setting to all 47 computers on the network at once.

Copy and paste the script below into the terminal, and press enter on your keyboard to view the result.

powershell
$computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name foreach ($computer in $computers) { Invoke-Command -ComputerName $computer -ScriptBlock { Set-ExecutionPolicy RemoteSigned -Force } }
What Just Happened — Line by Line
Get-ADComputer
Queries Active Directory and pulls every computer on the network. The administrator does not type a single machine name. The script finds them all automatically.
Foreach
Loops through every machine returned by Active Directory. One machine or one thousand — the loop handles them all the same way.
Invoke-Command
Reaches out to each remote computer over the network and runs the script block on that machine — without the administrator ever leaving his workstation.
Set-ExecutionPolicy RemoteSigned
Tells each machine to allow PowerShell scripts to run automatically. From this point forward, every script that IT pushes to Mach Regional Medical Center computers will execute without being blocked.
All 47 machines now run every script automatically. No one walked the floor. No one touched a single workstation. What would have taken hours was done in seconds from one terminal.

Reading a Patient Census CSV and Counting by Department
The admissions team at Mach Regional Medical Center needs a daily count of patients in each department. Instead of opening a spreadsheet manually, a Python script reads the census CSV file and outputs a department summary automatically.
Copy and paste the script below into the terminal, and press Enter on your keyboard to view the result.
python
import csv from collections import Counter filename = "patient_census.csv" departments = [] with open(filename, newline="") as f: reader = csv.DictReader(f) for row in reader: departments.append(row["Department"]) counts = Counter(departments) print("Patient Census by Department") print("-" * 32) for dept, count in sorted(counts.items()): print(f" {dept:<20} {count} patients")
What just happened — line by line
import csv
Loads Python's built-in CSV module. No installation required — it ships with every Python installation. One line gives you full CSV parsing capability.
Counter()
Counts occurrences of each department name automatically. Without Counter, you would need to write a loop, check if the key exists, and increment manually. Python collapses that into one call.
f-string
Formats each output line with aligned columns. The :<20 pads the department name to 20 characters so the patient counts line up cleanly.
Python strengths here
  • Built-in CSV module — no extra tools needed
  • Counter handles all counting logic in one line
  • Readable syntax — easy to audit and maintain
Python limitations here
  • Slower than compiled languages on very large files
  • Requires Python to be installed on the machine
  • Bash could do this with awk for simpler counts
For structured data like CSV files, Python is the right tool. Its built-in libraries handle parsing, counting, and formatting in a fraction of the code that PowerShell or Bash would require.
