Lesson 8: Configuring Your Browser for Safety

Core Principle: A misconfigured browser is a security liability. Update it regularly, disable Autofill, enable security settings, and manage cookies properly to minimize risk.

📋 Scenario

A user visits a malicious site that harvests the credit card and address info saved in their browser's Autofill. They also have dozens of third-party tracking cookies and outdated browser extensions. Review what browser security configurations should have been in place.

⚙️ Browser Security Checklist

🔄 Keep Browser Updated

  • Enable automatic updates
  • Update add-ons and extensions too
  • Patches fix known security vulnerabilities
  • Outdated browsers are primary attack vectors

🚫 Disable Autofill

  • Autofill stores passwords, addresses, CC numbers
  • Malicious sites can harvest saved form data
  • Use a dedicated password manager instead
  • Disable in Settings → Autofill settings

🔒 Enable Security Settings

  • Enable pop-up blocking
  • Enable safe browsing / phishing protection
  • Block dangerous downloads
  • Review and audit installed extensions

🍪 Manage Cookies

  • Block third-party tracking cookies
  • Clear cookies regularly
  • Use private/incognito for sensitive browsing
  • Don't accept all cookies by default

✅ Concept Check

Question: Why should browser Autofill be DISABLED as a security best practice?

A) Autofill slows down the browser significantly
B) Autofill only works on mobile devices
C) Malicious websites can potentially harvest saved form data including passwords and credit card numbers
D) Autofill prevents cookies from working properly
Explanation: Autofill stores sensitive data (passwords, addresses, credit card numbers) that malicious scripts on compromised websites can harvest. Using a dedicated password manager is a more secure alternative that doesn't expose data to browser-based attacks.

✅ Hardening Actions Explained
Apply OS patches: Closes known vulnerabilities — unpatched systems are the #1 target in most attacks.

Change default password: Default credentials like "admin/admin" are publicly documented; attackers try them first.

Disable AutoPlay: Stops removable media from automatically running code — prevents the "parking lot USB drop" attack.

Enable firewall: Blocks unauthorized inbound/outbound connections at the host level.

Install antimalware: Provides detection and removal of malware that gets through other defenses.

Remove unused software/services: Every extra program is a potential attack surface — minimize exposure.

Research software sources: Prevents installation of malware disguised as legitimate software.
❌ Non-Hardening Actions
Change desktop wallpaper: Purely cosmetic — zero security impact.

Install additional media player: Adding more software increases attack surface — opposite of hardening.

Leave Telnet enabled: Telnet transmits credentials in plaintext and is a known attack vector. Hardening requires disabling unused, insecure services.
💡 Pro Tip: The core principle of device hardening is attack surface reduction — remove, disable, change, and protect everything a threat actor could exploit.

Install & Configure Anti-Malware
PROTECTION
Objective
Deploy and configure an endpoint anti-malware solution to detect, quarantine, and remove malicious software. This is a foundational layer of host-based security.
Scenario
A new workstation has been added to the network. Before deploying it, ensure it has active, up-to-date anti-malware protection configured with real-time scanning enabled.
Procedure
  • 01
    Open Windows Security — Press Win + I → Settings → Privacy & Security → Windows Security.
  • 02
    Review Virus & threat protection — Confirm Real-time protection is On.
  • 03
    Update definitions — Click Check for updates. Definition files must be current.
  • 04
    Enable Cloud-delivered protection — Toggle On. Sends suspicious samples to Microsoft for real-time analysis.
  • 05
    Verify Quarantine — Under Protection history, review any quarantined items.
⚠️ Anti-malware is only effective when its definition database is current. A signature that is weeks old may miss recently discovered malware variants.
⚡ Windows Security — Virus & Threat Protection

Enable all required protections. Avoid settings that reduce scanning coverage.
