Utility Software: Programs designed to help manage, maintain, and protect your computer system. These tools perform specific tasks that keep your system running smoothly and securely.

B) PDF reader

Explanation: Compression utilities (like WinZip or 7-Zip) are designed to compress multiple files into a single archive file (like .zip or .rar), reducing file size and making it easier to share multiple files as one package.

📋 Executable Extensions Reference Guide — Study this before you begin ▲ Hide Table
Extension Platform Type Key Characteristics
.app macOS Application Bundle A directory (folder) that macOS presents as a single clickable item; contains the compiled binary, resources, and metadata inside a structured bundle; dragging to Applications folder installs it; cannot run on Windows
.bat Windows Batch Script A plain text file containing a sequence of Windows Command Prompt (CMD) commands executed line by line; no compilation required; source code is visible in any text editor; used for automation, deployment scripts, and repetitive task execution
.exe Windows Compiled Executable A compiled binary file that Windows executes directly; can be a standalone application or an installation wizard; the most common Windows executable format; source code is not visible; may trigger a UAC prompt if it requests elevated privileges
.msi Windows Windows Installer Package A structured installation package used by the Windows Installer service; supports rollback on failure, repair, logging, and silent/unattended installation via command-line switches; the standard format for enterprise software deployment via Group Policy or SCCM
.scexe Windows (vendor-specific) Self-Contained Executable A single self-extracting executable that bundles a compressed payload (drivers, firmware, utilities) and extracts it at runtime before executing; commonly used by HP for hardware driver and firmware updates; simplifies distribution as one file requires no separate extraction step

💡 Tip: Know the platform first — .app is macOS only; .bat, .exe, .msi, and .scexe are Windows. Then know the type — .exe is a compiled binary; .bat is a readable script; .msi is a structured installer with rollback and enterprise deployment support; .scexe is a self-contained single-file package that extracts its own payload. Never run any of these from an untrusted source — all executable types can carry malware.

Performance-Based Question

A software vendor releases a small utility that checks whether a workstation meets the hardware requirements for an upcoming application deployment. The utility is a single compiled file — no installation required, no setup wizard, no supporting files needed. A technician simply copies it to any Windows workstation, double-clicks it, and the tool runs its checks and displays results immediately.

Hint: Focus on “no installation required, no setup wizard, no supporting files needed.”
🔍 .exe — Portable Standalone Utility 
Platform: Windows
Type: Compiled binary — portable; no installation or Windows Installer service required
In this scenario: Single file, no install needed, runs anywhere = portable .exe.
Key distinction: A portable .exe needs no installation. A .msi always requires the Windows Installer service. A .bat needs no compilation but requires CMD commands. Only .exe can be a portable compiled tool.
