Lesson 7: LAN vs. WAN — Network Scope

Network Scope: Networks are classified by their geographic coverage. A Local Area Network (LAN) covers a small area like a building, while a Wide Area Network (WAN) spans larger distances, often connecting multiple LANs together.

📋 Scenario

Your company has offices in Houston, Dallas, and Austin. Each office has its own network of computers. You need to connect all three offices so employees can share resources and communicate. What type of networks are you dealing with?

LAN vs. WAN Comparison

FeatureLAN (Local Area Network)WAN (Wide Area Network)

Typical SizeUp to a few kilometersUnlimited — can span continents

💡 Concept Check

You connect 50 computers together in a single office building using Ethernet switches. What type of network is this?

WAN — because it has many computers
LAN — because it's confined to a small geographic area
Both LAN and WAN
Neither — this is a peer-to-peer network
Correct! This is a LAN. Network type is determined by geographic scope, not the number of devices. Since all 50 computers are in one building, it's a Local Area Network. A WAN would connect networks across different geographic locations.

Business Requirements
Distributed workforce accessing files from multiple countries
Five-person dental office wanting plug-and-play centralized storage
Large hospital with dedicated IT staff and strict regulatory compliance
✅ Why Correct
File Server is correct for the hospital: runs on a dedicated computer with server OS, allows granular permission control, meets compliance needs — but requires dedicated IT staff.

NAS fits the dental office: a plug-and-play storage appliance (think Synology or QNAP) that connects directly to the local network with minimal IT expertise and lower cost than a full server.

Cloud Storage (Google Drive, OneDrive, Dropbox) is the only option that works for a geographically dispersed workforce — accessible from anywhere with an internet connection.
❌ Common Mistakes
Confusing NAS with a File Server: NAS is a dedicated appliance — simpler but less powerful. A File Server is a full computer running server OS software with IT administrative overhead.

Assigning Cloud to compliance-heavy orgs: Cloud adds subscription costs and data sovereignty concerns. Not ideal where strict regulatory compliance is required.
💡 Pro Tip: "Remote access / anywhere" → Cloud · "Plug-and-play / small office" → NAS · "Dedicated IT / large org / compliance" → File Server.

A hotel has a single wired network running to each floor's telecommunications closet. The hotel wants every room on the floor to have wireless internet without running individual cables to each room.

📖Access Point — Enterprise Hotel Deployment
Function: Converts the wired network connection in the closet into a wireless signal for each room.

In this scenario: One AP per room or hallway, all wired back to the same switch, gives guests seamless Wi-Fi coverage throughout the floor.

Key distinction: The AP does not perform routing or DHCP — those services run from centralized infrastructure.

A coffee shop provides free Wi-Fi. When customers connect, they automatically receive an IP address (192.168.100.x), subnet mask (255.255.255.0), gateway (192.168.100.1), and DNS servers without any manual configuration.
What protocol provides this automatic network configuration?
ARP
DNS
HTTP
DHCP
Correct Answer: DHCP (Dynamic Host Configuration Protocol)

Explanation: DHCP is specifically designed for automatic network configuration: (1) Device connects to network, (2) Broadcasts DHCP Discover message, (3) DHCP server responds with DHCP Offer (IP address, subnet, gateway, DNS), (4) Device accepts with DHCP Request, (5) Server confirms with DHCP ACK. DHCP provides: IP address from pool, subnet mask (defines network size), default gateway (router for internet access), DNS servers (for name resolution), and lease time (how long IP is valid). Without DHCP, every device would need manual configuration — imagine configuring hundreds of phones and laptops manually! Public Wi-Fi networks require DHCP for user convenience.
